# Tech Stack

Appota HRM sử dụng các công nghệ hiện đại, tối ưu cho **DX** (Developer Experience) và **performance**.

## Frontend

| Layer | Công nghệ | Lý do |
| --- | --- | --- |
| Framework | React 18 | Stable, ecosystem rộng |
| Language | TypeScript | Type safety, IDE support |
| Build | Vite (rolldown-vite) | Build nhanh, HMR tức thì |
| Styling | Tailwind CSS v3 | Utility-first, tree-shakeable |
| Components | shadcn/ui (Radix UI) | Accessible, customizable |
| Routing | React Router v7 | Data router, type-safe |
| Forms | React Hook Form \+ Zod | Performance \+ validation |
| Charts | Recharts | Composable, declarative |
| Icons | lucide-react | Tree-shakeable, đẹp |
| Linter | Biome | Nhanh hơn ESLint |

## Backend (Optional)

| Service | Mục đích |
| --- | --- |
| Supabase | Postgres database, Auth, Storage, Realtime |

<Note>
  Hệ thống **chưa kết nối** Supabase trong phiên bản hiện tại. Tất cả logic được test với mock data và sẵn sàng tích hợp.
</Note>

## Tại sao không dùng Redux / Zustand?

Hệ thống chọn **Context API** vì:

- Bundle size nhỏ hơn
- Logic state đơn giản (HRMContext chỉ quản lý StaffProfile\[\])
- Tránh over-engineering
- Dễ onboard developer mới

Khi nào nên cân nhắc Zustand/Redux:

- State updates \> 100/giây
- Nhiều components subscribe vào state phức tạp
- Cần time-travel debugging

## Tại sao không dùng Next.js?

- Hệ thống là **internal tool** — không cần SEO
- Toàn bộ logic ở phía client (mock data)
- Bundle SPA đơn giản, không cần SSR/ISR
- Vite \+ React Router cho trải nghiệm dev tốt hơn

## Performance Optimizations

- **Code splitting** tự động theo route
- **Tree-shaking** Tailwind CSS chỉ giữ classes sử dụng
- **Image optimization** qua Vite asset pipeline
- **Lazy loading** components nặng (Recharts, video-react)
- **CSS variables** thay vì inline colors → giảm class trùng lặp
