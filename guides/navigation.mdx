# Điều hướng & Layout

Appota HRM sử dụng **AppLayout** làm shell chính, gồm header, sidebar và `<Outlet />` render nội dung route. Breadcrumb được tự động render cho các trang drill-down (Staff 4 cấp, Workforce Department).

## Cấu trúc AppLayout

```text
┌─────────────────────────────────────────────┐
│  Header (logo, search, theme, user menu)    │
├──────┬──────────────────────────────────────┤
│      │                                       │
│ Side │        <Outlet />                     │
│ bar  │        (Page content)                 │
│      │                                       │
└──────┴──────────────────────────────────────┘
```

## Breadcrumb cho Staff 4 cấp

Khi duyệt nhân viên, breadcrumb cập nhật theo từng cấp:

```text
Appota Group
   └── Danh sách Công ty
          └── Appota Game
                 └── Engineering
                        └── Danh sách Nhân viên
                               └── Nguyen Van A (Staff Detail)
```

Click vào bất kỳ cấp nào trên breadcrumb để quay lại cấp đó với transition mượt.

## Routing

Tất cả route được khai báo trong `src/routes.tsx` theo kiểu `RouteConfig`:

```typescript title="src/routes.tsx"
import type { RouteConfig } from './types';

export const routes: RouteConfig[] = [
  { path: '/', element: <DashboardPage /> },
  { path: '/staff', element: <StaffListPage /> },
  { path: '/staff/:id', element: <StaffDetailPage /> },
  { path: '/onboarding', element: <OnboardingPage /> },
  { path: '/onboarding/form', element: <OnboardingFormPage /> },
  { path: '/hr-review', element: <HRPendingPage /> },
  { path: '/hr-review/:id', element: <HRReviewDetailPage /> },
  // ...
];
```

<Tip>
  Để thêm route mới: tạo page trong `src/pages/`, import vào `routes.tsx`. Route tự động render trong `AppLayout`.
</Tip>

## Route Guard

Hệ thống sử dụng `<RouteGuard>` để chặn truy cập khi user không có quyền:

```tsx
<RouteGuard requires={['view-staff']}>
  <StaffListPage />
</RouteGuard>
```

Khi vi phạm, hệ thống hiển thị **"Access denied"** hoặc redirect về `/`.
