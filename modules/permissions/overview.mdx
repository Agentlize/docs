# Permissions

**Route:** `/permissions`

Hệ thống phân quyền theo mô hình **RBAC** (Role-Based Access Control) với 9 role, 24 permission và 25-30 policy.

## Vai trò

| Role | Phạm vi |
| --- | --- |
| **Super Admin** | Toàn hệ thống |
| **HR Manager** | Toàn hệ thống |
| **Department Head** | Phòng ban của mình |
| **Interviewer** | Interview được phân công |
| **Recruiter** | Toàn hệ thống (CV/JD Pool) |
| **Leader** | Phòng ban của mình |
| **Trainer** | Khóa học của mình |
| **Employee** | Cá nhân |
| **Guest** | Không truy cập |

## Permission

Hệ thống có **24 permission** thuộc các resource:

- Staff (View, Edit, Delete, Approve Onboarding)
- Interview (View, Edit Score)
- Workforce Planning (View, Edit)
- CV Pool (View, Edit)
- JD Pool (View, Edit)
- Audit Trail (View)
- Permissions (Manage)
- Recruitment Orders (Create, Approve)
- Reports (View)
- LMS (View, Manage Courses, Materials, Quizzes, View Progress, Manage Training Plans, View Training Reports)

## Policy

```typescript
interface Policy {
  id: string;
  role: string;
  resource: string;
  action: string;
  condition?: string;
}
```

Ví dụ policy:

| Role | Resource | Action | Condition |
| --- | --- | --- | --- |
| Leader | Recruitment Orders | Create | — |
| Department Head | Staff | View | `departmentId = currentUser.departmentId` |
| HR Manager | CV Pool | Edit | — |

## Department Boundary

User chỉ xem được dữ liệu của **phòng ban mình** (trừ Super Admin và HR Manager).

## Permission Boundary

<Warning>
  - **Department Head** chỉ xem data của **department mình**.
  - **Employee** chỉ xem profile của **mình**.
  - **HR Manager** và **Super Admin** xem **toàn bộ data**.
</Warning>

Khi vi phạm boundary: hiển thị **"Access denied"** hoặc redirect về `/`.

Xem chi tiết tại [Permission Boundary](/business-rules/permission-boundary).
